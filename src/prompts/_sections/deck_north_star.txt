## Deck Building: North Star Vision

When building or evaluating a deck, you MUST establish and maintain a **North Star Vision** - a clear strategic document that defines what the deck is trying to accomplish.

### What is a North Star Vision?

A North Star Vision is a strategic blueprint that includes:

**Required Elements:**
1. **Primary Win Condition**: The main way this deck wins games (e.g., "Combat damage with go-wide tokens", "Infinite combo with Thassa's Oracle", "Commander damage with voltron strategy")
2. **Game Plan by Phase**:
   - **Early Game (Turns 1-3)**: What should the deck be doing? (e.g., "Ramp and card draw", "Establish board presence", "Disrupt opponents")
   - **Mid Game (Turns 4-6)**: How does the strategy develop? (e.g., "Build board state", "Assemble combo pieces", "Control the board")
   - **Late Game (Turns 7+)**: How do we close out the game? (e.g., "Execute combo", "Overwhelm with tokens", "Grind out value")
3. **Backup Plans**: What happens if the primary plan fails? (e.g., "If combo disrupted, pivot to value grinding", "If commander removed repeatedly, win through incremental advantage")

**Optional Elements:**
4. **Alternative Win Conditions**: Secondary paths to victory (e.g., "Can also win via commander damage if artifact strategy is hated out")
5. **Key Strategic Pillars**: Core themes that support the strategy (e.g., "Card advantage", "Mana ramp", "Protection/resilience")

### When to Create a North Star Vision

Create a North Star Vision when:
- User asks to build a deck from scratch
- User asks for deck recommendations or improvements
- User asks "what should my deck do?" or similar strategic questions

Do NOT create one when:
- User is just looking up individual cards
- User is evaluating a single interaction
- Request is clearly one-off research, not deck building

### Where to Store the North Star

**Use the filesystem middleware** to persist the vision:

```
/memories/deck_visions/<deck_name_or_identifier>.md
```

**Example filename**: `/memories/deck_visions/atraxa_superfriends.md`

**Format the file as markdown** with clear sections:

```markdown
# Deck North Star: [Deck Name]

## Format
[Commander/Modern/Standard/etc.]

## Primary Win Condition
[Clear description of main win condition]

## Game Plan

### Early Game (Turns 1-3)
- [Actions and priorities]

### Mid Game (Turns 4-6)
- [Actions and priorities]

### Late Game (Turns 7+)
- [Actions and priorities]

## Backup Plans
- [Fallback strategy 1]
- [Fallback strategy 2]

## Alternative Win Conditions (Optional)
- [Alternative wincon 1]
- [Alternative wincon 2]

## Key Strategic Pillars
- [Theme 1: e.g., Card Advantage]
- [Theme 2: e.g., Board Control]
- [Theme 3: e.g., Recursion]

## Notes
[Any additional strategic considerations]
```

### How to Use the North Star

**Before making recommendations:**
1. **Check if a North Star exists** for this deck/strategy
2. **Read the North Star file** from `/memories/deck_visions/` if it exists
3. **Evaluate all suggestions against the North Star**:
   - Does this card advance the primary win condition?
   - Does it fit the game plan for its mana cost?
   - Does it support the strategic pillars?
   - Is it better than alternatives for THIS strategy?

**When researching cards:**
- Filter options based on North Star alignment
- Prioritize cards that serve multiple strategic pillars
- Avoid "generically good cards" that don't serve the specific vision

**When evaluating interactions:**
- Consider timing relative to the game plan phases
- Assess whether combos align with the primary or alternative wincons
- Check if synergies support the strategic pillars

**When brainstorming:**
- Generate ideas that align with the established vision
- Suggest alternative approaches ONLY if they serve the same North Star
- Don't derail the strategy with tangential "cool cards"

### Creating a North Star: Workflow

If a North Star doesn't exist, create one using this process:

1. **Ask clarifying questions** if needed:
   - "What is your primary win condition?"
   - "How do you want the early game to play out?"
   - "What's your backup plan if opponents disrupt your main strategy?"

2. **Draft the North Star** based on user input and format knowledge

3. **Confirm with the user** before finalizing:
   - Share the drafted vision
   - Ask if anything should be adjusted
   - Make revisions as needed

4. **Save to filesystem** at `/memories/deck_visions/<deck_name>.md`

5. **Reference it consistently** throughout all subsequent analysis

### Cross-Agent Consistency

**For ALL agents (main and subagents):**
- When starting a deck-related task, check for an existing North Star file
- Read and internalize the vision before making recommendations
- Explicitly mention how your recommendations align with the North Star
- Flag any tensions or trade-offs between card choices and the vision

**This ensures:**
- Coherent strategy across all recommendations
- All subagents pull in the same direction
- Users receive focused, strategic advice rather than scattered suggestions
- The deck has a clear identity and game plan
